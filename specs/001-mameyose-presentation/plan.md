# Implementation Plan: 豆寄席プレゼンテーション資料作成

**Branch**: `001-mameyose-presentation` | **Date**: 2026-03-05 | **Spec**: [001-mameyose-presentation/spec.md](spec.md)

## Summary

本計画書は、「豆寄席」で発表予定の90分枠（発表80分＋Q&A10分）のプレゼンテーションスライド作成のための構成案を示すものです。
スクラムマスターを主なターゲットとし、AIを活用した経験的プロセス制御（透明性・検査・適応）の強化手段を具体的な事例（Jira×GAS×AIでの可視化、AIによるレトロスペクティブ評価）を通して解説します。また、最後に当スライドの作成自体をAIエージェント（Antigravity）やGitHub Spec Kit、Marpを用いて行った裏側（メイキング）を紹介します。

## Technical Context

**形式**: `Marp for VS Code` (MarkdownからスライドPDF/HTMLを出力)
**デザインシステム**: GitHub Primer CSSのカラースキームと構成に準拠、Octicons使用
**ベースカラー**: 少し濃い紫色 (`#EDE1F8` など)  
**スライドテンプレ機能**: 右上に会社ロゴ（`mamezo.png`）固定配置、フッターにページ番号のみ  
**フォント**: ユニバーサルデザインフォント (`BIZ UDPGothic` 等) ベース。最低 24pt を確保。
**使用生成AIツール**: Antigravity, GitHub Spec Kit
**CI/CD**: GitHub Actionsを用いたMarpスライドのPDF/HTML自動ビルドとGitHub Pagesへの自動デプロイ

## タイムライン＆構成案（全80分 / スライド枚数: 約56枚）

1スライド1メッセージの原則で作成するため、テンポよくめくる箇所とじっくり解説する箇所で緩急をつけます。

### 1. イントロダクションと自己紹介 (最大10分 / スライド9枚)
- **ファイル**: `00_intro.md`, `01_self_intro.md`
- タイトルロゴ（`images/title.png`）を使った表紙
- テーマの背景と本日のゴール（透明性・検査適応・SM像の3本柱）
- **第0部：自己紹介** 
  - プロフィール、これまでのアジャイル/スクラムマスターの経験、ルービックキューブ・メガミンクス競技
  - 今回の豆寄席について（開発者ではなくスクラムマスターはどうAIを使うべきか）

### 2. 第1部：スクラムとAIの導入 (約10分 / スライド4枚)
- **ファイル**: `02_scrum_ai.md`
- スクラムガイド拡張パック（2025.6版）における「人工知能」の定義
- AIによる経験的プロセス制御の改善の可能性と、本日のフォーカスへの接続

### 3. 第2部：透明性の強化 (約25分 / スライド18枚)
- **ファイル**: `03_transparency.md`
- 透明性の本質（事実に基づく議論の土台）
- 実践例1: Jira × GAS × AI による Four Keys の可視化
- 実践例2: リリース必達案件における「加速（着地予測）」の可視化
- アウトカム: 体感からデータ（事実）に基づく意思決定への適応と、スクラムマスターが最初期にツールを作る意義

### 4. 第3部：検査・適応の強化 (約20分 / スライド9枚)
- **ファイル**: `04_inspection.md`
- 検査と適応の障壁（マンネリ・遠慮等）
- 実践: AIによるレトロスペクティブ（文字起こし等）の定量評価とフィードバック
  - 10項目の評価基準とスコアリング
- チームに起きた「適応」の実例（5W1Hの徹底、サンクスカード等）と、今後の展望（デイリースクラムへの展開）

### 5. まとめ＆おまけ (最大15分 / スライド16枚)
