# Feature Specification: 豆寄席プレゼンテーション資料作成

**Feature Branch**: `001-mameyose-presentation`  
**Created**: 2026-03-05
**Status**: Draft  

## User Scenarios & Testing *(mandatory)*

### User Story 1 - スクラムマスターへのAI活用事例の紹介と実践の促進 (Priority: P1)

参加者（AI活用に興味のあるスクラムマスターやプロセス改善層）が、AIを単なるコーディングアシスタントとしてではなく、スクラムイベントを支援する実務的パートナーとして活用する手段を理解し、翌日から実践できる具体的なヒントを得る。

**Why this priority**: 本プレゼンテーションの中核となる目的であり、もっとも高い価値を提供するため。

**Independent Test**:
- プレゼン後に参加者が「透明性の向上」および「検査・適応の促進」においてAIをどう使うかのアイデアを説明できるか。
- 各活用事例（Jira×GAS×AIでの可視化、レトロスペクティブのAI評価）の実践手順が明確に伝わるか。

**Acceptance Scenarios**:

1. **Given** 参加者がAIのスクラム活用に漠然とした課題感を持っている状態、**When** プレゼンテーション（導入〜透明性〜検査・適応）を聴講する、**Then** 実務で使えるAI活用のアプローチを3つ以上言語化できるようになる。
2. **Given** 質疑応答の時間が設けられている状態、**When** 参加者が自チームの課題を質問する、**Then** 登壇者が紹介した事例を元に解決のヒントを提示できる。

---

### Edge Cases

- 質疑応答で想定外のAI技術に関する質問が出た場合はどう対応するか？（→あくまでスクラムの実務適用事例に絞り、高度な技術論には深入りしない方針を持つ）
- デモやツール紹介で画面が見えづらい場合はどうするか？（→図解や大きなフォントを活用し、Constitutionのルールに従って視認性を確保する）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: プレゼンテーション全体の所要時間は90分とする。
- **FR-002**: 時間配分は、発表本編を80分、質疑応答（Q&A）を最後の10分とする。途中の休憩やディスカッションは挟まない。
- **FR-003**: ターゲット聴衆である「AI活用に興味のあるスクラムマスターやプロセス改善をしたい人」に合わせた平易で実践的なトーンとする。
- **FR-004**: 発表時間（80分）に余裕があるため、プレゼンテーション冒頭の自己紹介セクションを厚めにとり、登壇者の背景や普段の業務、スクラム・AIとの関わりを丁寧に伝える。
- **FR-005**: 以下の3つの記事（アジェンダ）をベースに一連のストーリーとして構成する。
  1. 第1部：スクラムマスターのAI活用を考える - 導入（スクラムガイド拡張パックとAIの全体像）
  2. 第2部：透明性（Jira×GAS×AIによる可視化指標 FourKeys等 のツール作成事例）
  3. 第3部：検査・適応（AIによるレトロスペクティブの定量評価とデイリースクラムへの応用）
- **FR-006**: プレゼンテーションの最後（まとめの後、Q&Aの前）に、「このスライド自体をどのように作成したか（メイキング）」として、**Antigravity、GitHub Spec Kit、Marp** を用いたスライド作成の裏側を紹介するセクションを設ける。
- **FR-007**: 既存の `constitution.md` のデザインルール（Marp形式、Primer CSS/Octicons、薄い紫ベース、ロゴ右上固定、フォントルール）を遵守して作成する。

### Key Entities 

- **Presentation Slide**: Marp拡張機能を活用したMarkdownドキュメント（1スライド1メッセージの原則で構築）。
- **Source Articles**: baseとなる3つのMarkdown記事内容。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 発表内容を80分のタイムボックス内に収めることができる。（リハーサルで実証）
- **SC-002**: 導入、透明性、検査・適応の3部構成間のつながりが違和感なくストーリーとして繋がっている。
- **SC-003**: 提供された3つの記事の重要ポイントが欠落することなく、スライド形式に合わせて適切に要約・図解案が盛り込まれている。
- **SC-004**: 定められたデザイン規約（constitution.md）を満たしたMarpスライドとしてプレビュー出力がエラーなく行える。
